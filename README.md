# 🛡️ Bugout Monitor

**AI-powered disaster preparedness dashboard with real-time OSINT threat monitoring.**

An open-source tool that monitors global threats — solar storms, nuclear tensions, severe weather, economic instability — and presents them in a single dashboard alongside your personal bugout & disaster plan. Configurable for any location.

## Features

- **🎯 Threat Dashboard** — AI-analyzed threat levels (1-10) for Solar/Carrington events, Nuclear threats, Severe Weather, and an Overall composite score
- **📡 OSINT Intelligence Feed** — Real-time data from 8 free APIs (NOAA, NASA, NWS, GDELT, FRED, USDM, Arms Control RSS)
- **📖 Diamond Collapse Index** — Jared Diamond's 5-factor framework for societal collapse, scored and tracked
- **🎒 Equipment Inventory** — Interactive gear tracker with add/edit/delete, saved in your browser
- **📍 Rally Points** — Click-to-edit meeting locations for your team
- **🗺️ Bugout Plan Template** — Full 12-section disaster plan with routes, comms, scenarios, and checklists
- **⏰ Daily Automation** — Windows Task Scheduler or cron job runs the monitor daily
- **🖨️ Print-Ready** — Clean print layout for hard-copy backup

## Quick Start

```bash
# Clone the repo
git clone https://github.com/yourname/bugout-monitor.git
cd bugout-monitor

# Install dependencies
npm install

# Run the interactive setup wizard
node setup.js

# Open the dashboard
open index.html   # macOS
start index.html  # Windows
```

The setup wizard will:
1. Ask for your location (auto-detects county, FIPS code, NWS zone via API)
2. Configure nearby military/nuclear targets for threat assessment
3. Set up your LLM provider (OpenRouter recommended, or rule-based fallback)
4. Generate your personal `user-config.json` and `.env` files
5. Run your first threat scan

## Data Sources

All free, no paid APIs required.

| Source | Data | Auth |
|--------|------|------|
| [NOAA SWPC](https://services.swpc.noaa.gov) | Kp index, geomagnetic storm scale, solar flare probabilities | None |
| [NASA DONKI](https://api.nasa.gov) | CME events, solar flares, geomagnetic storms | Free API key |
| [NWS API](https://api.weather.gov) | Severe weather alerts for your county | None |
| [GDELT](https://api.gdeltproject.org) | Global news monitoring — nuclear, military, unrest keywords | None |
| [Arms Control Assoc](https://armscontrol.org) | Nuclear policy headlines via RSS | None |
| [IAEA](https://iaea.org) | Nuclear agency news via RSS | None |
| [US Drought Monitor](https://droughtmonitor.unl.edu) | Drought severity for your county | None |
| [FRED](https://fred.stlouisfed.org) | Economic stress indicators — CPI, unemployment, consumer sentiment | None |

## LLM Options

The monitor sends collected OSINT data to an LLM for nuanced threat analysis:

| Provider | Cost | Setup |
|----------|------|-------|
| **OpenRouter** (recommended) | ~$0.01/scan | Get key at [openrouter.ai/keys](https://openrouter.ai/keys) |
| **No LLM** (fallback) | Free | Rule-based thresholds — still works, less nuanced |

The LLM analyzes all 8 data sources together and produces calibrated threat levels with reasoning. Without an LLM, the system uses simple thresholds (e.g., Kp > 5 = elevated solar threat).

## Daily Automation

### Windows (Task Scheduler)

```bash
# Run the included batch file as Administrator
scripts\install-task-win.bat
```

This creates a daily task at 6:00 AM.

### Linux / macOS (cron)

```bash
# Add to crontab
chmod +x scripts/install-cron-linux.sh
./scripts/install-cron-linux.sh
```

### Manual Run

```bash
npm run monitor
```

## Configuration

All personal data lives in `user-config.json` (auto-generated by `setup.js`, never committed to git):

- **Location** — lat/lon, county, state, FIPS code, NWS zone
- **Nearby targets** — military bases, nuclear facilities for threat assessment
- **Routes** — your bugout routes with waypoints
- **Comms** — radio frequencies, Meshtastic channels
- **Rally points** — meeting locations (also editable in the UI)
- **LLM settings** — provider and model selection

Edit this file anytime to update your configuration, then run `npm run monitor` to refresh.

## Diamond Collapse Index

Based on Jared Diamond's "Collapse: How Societies Choose to Fail or Succeed" (2005), the monitor tracks five factors that historically precede societal collapse:

1. **Environmental Damage** — Drought, resource depletion, pollution
2. **Climate Change** — Weather extremes, agricultural disruption
3. **Hostile Neighbors** — Military tensions, geopolitical instability
4. **Trade / Supply Chain** — Economic stress, supply disruptions, inflation
5. **Society's Response** — Institutional capacity, political polarization

Each factor is scored 1-10 by the LLM based on current OSINT data, producing a composite collapse risk index.
